Sistema de Gerenciamento PQFL

Um breve resumo sobre o software para controle de qualidade do leite

Introdução

Este projeto tem como objetivo desenvolver um software para o gerenciamento do Plano de Qualificação de Fornecedores de Leite (PQFL), em conformidade com as exigências das Instruções Normativas 76 e 77 de 2018 do Ministério da Agricultura, Pecuária e Abastecimento (MAPA). O software está sendo desenvolvido por uma pequena equipe de 7 integrantes, composta por alunos e professores do Instituto Federal Fluminense (IFF) – Campus Bom Jesus do Itabapoana em parceria com um laticínio local, para coleta de informações para o software. O projeto é financiado por meio de bolsas do Conselho Nacional de Desenvolvimento Científico e Tecnológico (CNPq) e da Fundação Carlos Chagas Filho de Amparo à Pesquisa do Estado do Rio de Janeiro (Faperj).

A iniciativa visa automatizar processos essenciais no controle de qualidade do leite, como a gestão de visitas técnicas, avaliação de fornecedores, monitoramento de dados laboratoriais e conformidade regulatória. Através dessa automação, espera-se aumentar a eficiência operacional, garantindo a segurança e a qualidade dos produtos lácteos.

Objetivo

O objetivo geral deste projeto é desenvolver um software desktop especializado para o gerenciamento completo do Plano de Qualificação de Fornecedores de Leite (PQFL), com foco na automação e otimização dos processos de controle de qualidade e conformidade dos fornecedores de leite. O sistema está sendo projetado para atender às exigências das Instruções Normativas 76 e 77 de 2018 do Ministério da Agricultura, Pecuária e Abastecimento (MAPA), proporcionando uma solução eficiente para o acompanhamento e registro de visitas técnicas, análise de dados laboratoriais e gerenciamento de produtores, tanques e suas propriedades.

O software terá como função principal facilitar a coleta e análise de dados críticos relacionados à qualidade do leite, permitindo a geração automática de relatórios, gráficos e notificações para apoiar a tomada de decisões por parte dos gestores. Além disso, ele será capaz de identificar possíveis não conformidades, como níveis elevados de CPP (Contagem Padrão em Placas), sugerindo medidas corretivas, e permitindo um monitoramento contínuo e eficaz da produção.

Esse sistema buscará não só cumprir os requisitos regulatórios, mas também aumentar a eficiência operacional de laticínios e cooperativas ao centralizar todas as informações necessárias para a avaliação de fornecedores em uma única plataforma, simplificando o processo de gerenciamento e promovendo a melhoria contínua da qualidade do leite ao longo da cadeia produtiva.

Metodologia

O desenvolvimento do software de gerenciamento do Plano de Qualificação de Fornecedores de Leite (PQFL) está sendo conduzido com base na metodologia ágil Scrumban, que combina elementos das metodologias Scrum e Kanban ( tudo aplicado no Notion para melhor gerenciamento). Essa abordagem foi escolhida por sua flexibilidade e eficiência na gestão de projetos, permitindo a adaptação contínua do desenvolvimento às necessidades específicas do cliente e do mercado. A seguir, detalho as principais etapas e práticas adotadas no processo de desenvolvimento:

  • Levantamento de Requisitos: Identificação das necessidades da cooperativa através de entrevistas e questionários.
  • Design e Prototipagem: Criação de fluxogramas e protótipos de tela com ferramentas como LucidChart®.
  • Desenvolvimento: Implementação do sistema conforme os protótipos e ajustes contínuos na base de dados.
  • Testes e Validação: Realização de testes de usabilidade e performance, com feedback contínuo da equipe.
  • Documentação e Entrega: Elaboração de documentação técnica e entrega final, com registro no INPI.

Arquitetura do Sistema

O Sistema PQFL foi construído utilizando uma arquitetura baseada em microserviços. Cada componente do sistema, desde o frontend até o backend, trabalha em conjunto para oferecer uma experiência rápida e eficiente:

  • Frontend: O JFrame foi utilizado para a criação das interfaces gráficas, com o auxílio do WindowBuilder para agilizar e facilitar o desenvolvimento das telas de forma visual.
  • Backend: O backend foi desenvolvido utilizando Java, oferecendo robustez e segurança para o processamento de dados e regras de negócio.
  • Banco de Dados: O MySQL Server foi adotado como sistema de gerenciamento de banco de dados, com o suporte do MySQL Workbench para simplificar a modelagem e gerenciamento das estruturas de dados.

Tecnologias Utilizadas

  • Java: Linguagem de programação principal utilizada para o desenvolvimento do software desktop.
  • Eclipse IDE: Ambiente de desenvolvimento integrado (IDE) utilizado para programar o sistema em Java.
  • SQL Server: Utilizado para a inicialização e administração do banco de dados, garantindo o armazenamento e processamento eficiente das informações.
  • SQL Workbench: Ferramenta usada para modelagem e gerenciamento do banco de dados SQL, essencial para armazenar e organizar os dados do sistema.
  • WindowBuilder: Ferramenta integrada ao Eclipse, utilizada para criar e editar interfaces gráficas (GUI) de forma visual e simplificada.
  • Visual Studio Code: Editor de código leve e versátil utilizado para edição de scripts e arquivos auxiliares do projeto.
  • Notion: Ferramenta usada para a gestão de processos, organização de documentos e acompanhamento do progresso do projeto.
  • GitHub: Plataforma utilizada para controle de versão e repositório de código, permitindo colaboração eficiente e rastreamento de mudanças.
  • LucidChart: Utilizado para criar fluxogramas e diagramas, ajudando no design e prototipagem do sistema.
